### Task 1: Fix Windows User PATH Registry

**Problem:** User PATH in registry is `%PATH%;Python312\Scripts;npm`. The `%PATH%` self-reference duplicates Machine PATH, causes confusion, and breaks new shell sessions when tools are added. Several user-installed tool paths that exist on disk are missing from the registry.

**Files:**
- Modify: Windows Registry `HKCU\Environment\PATH`

**Step 1: Back up the current registry User PATH**

```powershell
# Run from PowerShell (via powershell.exe -File -)
$key = [Microsoft.Win32.Registry]::CurrentUser.OpenSubKey('Environment')
$raw = $key.GetValue('PATH', '', [Microsoft.Win32.RegistryValueOptions]::DoNotExpandEnvironmentNames)
$key.Close()
Write-Host "BACKUP: $raw"
```

Expected: `%PATH%;C:\Users\paulp\AppData\Local\Packages\PythonSoftwareFoundation.Python.3.12_qbz5n2kfra8p0\LocalCache\local-packages\Python312\Scripts;C:\Users\paulp\AppData\Roaming\npm`

**Step 2: Write the corrected User PATH to registry**

Remove `%PATH%;` prefix. Add all user-specific directories that exist on disk. Use `%USERPROFILE%` for portability where appropriate.

New value (each entry on its own line for readability — joined by `;` in registry):

```
%USERPROFILE%\.cargo\bin
%USERPROFILE%\AppData\Local\Microsoft\WindowsApps
%USERPROFILE%\AppData\Local\Programs\Microsoft VS Code\bin
%USERPROFILE%\AppData\Local\GitHubDesktop\bin
%USERPROFILE%\.cache\lm-studio\bin
%USERPROFILE%\AppData\Local\Programs\cursor\resources\app\bin
%USERPROFILE%\AppData\Roaming\npm
%USERPROFILE%\AppData\Local\Packages\PythonSoftwareFoundation.Python.3.12_qbz5n2kfra8p0\LocalCache\local-packages\Python312\Scripts
```

```powershell
$key = [Microsoft.Win32.Registry]::CurrentUser.OpenSubKey('Environment', $true)
$newPath = @(
    '%USERPROFILE%\.cargo\bin',
    '%USERPROFILE%\AppData\Local\Microsoft\WindowsApps',
    '%USERPROFILE%\AppData\Local\Programs\Microsoft VS Code\bin',
    '%USERPROFILE%\AppData\Local\GitHubDesktop\bin',
    '%USERPROFILE%\.cache\lm-studio\bin',
    '%USERPROFILE%\AppData\Local\Programs\cursor\resources\app\bin',
    '%USERPROFILE%\AppData\Roaming\npm',
    '%USERPROFILE%\AppData\Local\Packages\PythonSoftwareFoundation.Python.3.12_qbz5n2kfra8p0\LocalCache\local-packages\Python312\Scripts'
) -join ';'
$key.SetValue('PATH', $newPath, [Microsoft.Win32.RegistryValueKind]::ExpandString)
$key.Close()
Write-Host "User PATH updated ($($newPath.Length) chars)"
```

**Step 3: Broadcast environment change**

```powershell
Add-Type -Namespace Win32 -Name NativeMethods -MemberDefinition '[DllImport("user32.dll", SetLastError = true, CharSet = CharSet.Auto)] public static extern IntPtr SendMessageTimeout(IntPtr hWnd, uint Msg, UIntPtr wParam, string lParam, uint fuFlags, uint uTimeout, out UIntPtr lpdwResult);'
[Win32.NativeMethods]::SendMessageTimeout([IntPtr]0xffff, 0x1a, [UIntPtr]::Zero, 'Environment', 2, 5000, [ref]([UIntPtr]::Zero))
```

Expected: Returns `1` (success).

**Step 4: Verify from a fresh PowerShell**

```powershell
powershell.exe -NoProfile -Command "claude --version; code --version; cargo --version"
```

Expected: All three commands resolve.

**Step 5: Verify registry is clean**

```powershell
$key = [Microsoft.Win32.Registry]::CurrentUser.OpenSubKey('Environment')
$raw = $key.GetValue('PATH', '', [Microsoft.Win32.RegistryValueOptions]::DoNotExpandEnvironmentNames)
$key.Close()
# Confirm NO %PATH% self-reference
Write-Host "Contains %%PATH%%: $($raw.Contains('%PATH%'))"
Write-Host "Entry count: $(($raw -split ';').Count)"
```

Expected: `Contains %PATH%: False`, `Entry count: 8`
